A technical guide addressing subdomain takeover attacks, where threat actors exploit dangling DNS records to redirect traffic. It explains how the tactic works, identifies vulnerable AWS resources (S3, CloudFront, Elastic Beanstalk), and provides detection using AWS Config custom rules, along with prevention procedures, mitigation strategies, and an open-source reference implementation for identifying CNAME records pointing to deleted resources.
